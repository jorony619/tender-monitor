# 手把手部署指南：让招标监控网页自动更新

跟着下面的步骤做，全程不需要写代码、不需要用命令行，大约10-15分钟。

---

## 第一步：注册 GitHub 账号（如果还没有）

1. 打开 https://github.com/signup
2. 填邮箱、密码、用户名，按提示完成注册（会发一封验证邮件，去邮箱点一下确认）

已经有账号的话，直接登录 https://github.com 跳到第二步。

---

## 第二步：新建一个仓库（repository）存放网页

1. 登录后，右上角点 **+** → **New repository**
2. **Repository name** 填一个英文名字，比如 `tender-watch`
3. 选择 **Public**（公开仓库免费用GitHub Pages和Actions；如果要私有仓库也可以，功能一样，只是私有仓库的Actions分钟数有限额）
4. 其他保持默认，**不要**勾选 "Add a README file"（我们等下会自己上传文件）
5. 点绿色的 **Create repository**

创建完成后，会跳到一个空仓库页面，上面写着 "Quick setup"，先留着这个页面。

---

## 第三步：把文件上传上去

1. 在刚才那个空仓库页面，找到一行小字链接 **uploading an existing file**，点它
   （如果没看到，就点仓库页面里的 **Add file → Upload files**）
2. 把我打包给你的 `tender-watch-auto.zip` 在电脑上解压出来，会得到一个文件夹，里面有：
   ```
   index.html
   data.json
   README.md
   scripts/update_tenders.mjs
   .github/workflows/update-tenders.yml
   ```
3. **把这个文件夹里的所有文件和文件夹，直接拖拽到浏览器的上传区域**（包括隐藏的 `.github` 文件夹——
   如果你的电脑不显示以 `.` 开头的文件夹，见下方"常见问题1"）
4. 拖拽完成后，往下滚动，在 "Commit changes" 那里点绿色按钮 **Commit changes**

上传完成后，仓库首页应该能看到 `index.html`、`data.json`、`scripts`、`.github` 这些文件/文件夹。

> ⚠️ 最容易漏传的是 `.github/workflows/update-tenders.yml` 这个文件，因为它在两层隐藏文件夹里。
> 如果上传后仓库里没有 `.github` 文件夹，请单独把这个文件传一次：
> Add file → Create new file，文件名直接输入 `.github/workflows/update-tenders.yml`（GitHub会自动建文件夹），
> 把内容粘贴进去。

---

## 第四步：申请 Anthropic API Key

1. 打开 https://console.anthropic.com/ 并登录/注册
2. 左侧找到 **Settings → API Keys**（或者直接访问 https://console.anthropic.com/settings/keys）
3. 点 **Create Key**，起个名字比如 `tender-watch`，创建后会显示一长串以 `sk-ant-` 开头的字符串
4. **立刻复制保存好这串字符**——它只会完整显示这一次，关闭弹窗后就再也看不到完整内容了
5. 需要在账户里绑定付款方式/预充值一点余额（这个功能本身每天调用一次，成本非常低，
   但账户需要有可用额度API才会正常工作）

---

## 第五步：把 API Key 加到仓库的 Secret 里

这一步是让GitHub Actions能安全地使用你的API Key，而不会被公开看到。

1. 回到你的GitHub仓库页面，点顶部 **Settings**（仓库自己的设置，不是账号设置）
2. 左侧菜单找到 **Secrets and variables → Actions**
3. 点绿色按钮 **New repository secret**
4. **Name** 一栏，一字不差地填：`ANTHROPIC_API_KEY`
5. **Secret** 一栏，粘贴你刚才复制的 `sk-ant-...` 字符串
6. 点 **Add secret**

---

## 第六步：开启 GitHub Pages（让网页有一个网址）

1. 还是在仓库 **Settings** 里，左侧找到 **Pages**
2. **Source** 那里，从下拉菜单选择 **GitHub Actions**（不是 "Deploy from a branch"）
3. 不需要再点保存，选中即生效

---

## 第七步：手动跑一次，验证一切正常

1. 点仓库顶部的 **Actions** 标签
2. 左侧应该能看到一个叫 **Update tender data** 的workflow，点它
3. 右边点 **Run workflow** 下拉按钮 → 再点绿色的 **Run workflow** 按钮确认
4. 等几秒刷新页面，会出现一条正在跑的记录（黄色圆圈转圈），点进去可以实时看日志
5. 如果两个job（`refresh-data` 和 `deploy-pages`）最后都变成绿色✅，说明成功了
6. 回到 **Settings → Pages**，页面顶部会显示一个类似
   `https://你的用户名.github.io/tender-watch/` 的网址——这就是你以后随时打开看看的监控网页

---

## 第八步：以后怎么用

- **什么都不用做**：默认每天北京时间早上9点，它会自动跑一次，搜索、核实、更新数据，重新发布网页
- **想立刻手动刷新一次**：随时回到 Actions 标签页，点 **Run workflow**
- **想改成更频繁**（比如每6小时一次）：在仓库里打开
  `.github/workflows/update-tenders.yml`，点右上角铅笔图标编辑，
  把 `cron: "0 1 * * *"` 改成 `cron: "0 */6 * * *"`，提交保存即可
  （不建议太频繁，每次运行都会消耗一点API额度）

## 常见问题

**Q1：我的Mac/Windows不显示 `.github` 这种以点开头的文件夹，传不上去怎么办？**
Mac上在Finder按 `Cmd+Shift+.` 可以切换显示隐藏文件；Windows在"文件资源管理器-查看-隐藏的项目"打勾。
或者干脆用第三步末尾说的"单独创建文件"方法，把 `.github/workflows/update-tenders.yml` 手动建出来。

**Q2：Actions跑失败了（红色❌），怎么查原因？**
点进那次运行记录，点 **refresh-data** job，展开 "Run tender search & update data.json" 那一步，
里面的报错信息通常会直接告诉你原因，最常见的是：
- `Missing ANTHROPIC_API_KEY` → 第五步的secret名字打错了或没保存成功，检查一下
- `401` 或 `authentication_error` → API Key不对或者账户没有可用额度，回Anthropic Console检查
- `insufficient_quota` / 余额不足 → 需要去Anthropic Console充值

**Q3：这样每天要花多少钱？**
一次调用大概是几千到一万多token（含联网搜索），按Sonnet模型的价格算，每天一次的成本通常是几美分到一两毛钱人民币的量级，一个月下来是很小的数目。可以在Anthropic Console的Usage页面实时看到花费。

**Q4：我想暂停自动化，但网页还想留着**
仓库 Settings → Actions → General，选择 "Disable Actions"，定时任务就不会再跑了，
网页依然能正常打开（显示最后一次更新的数据）。

**Q5：数据更新后，网页上的"上次核查"时间没变？**
浏览器有缓存，强制刷新一下（Windows: `Ctrl+F5`，Mac: `Cmd+Shift+R`）。

**Q6：以后想加字段、改样式，怎么办？**
把修改后的文件重新上传覆盖（Add file → Upload files，同名文件会提示覆盖），
或者把改动需求直接告诉我，我帮你改好新版本文件，你只要重新上传即可。
